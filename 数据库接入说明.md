# 数据库接入说明：使用 MySQL 保存 Connector 和文件信息

## 1. 数据库用途

当前项目支持把 Connector 相关信息保存到 MySQL / MariaDB 中，方便你用 Navicat 查看。

数据库保存的是元数据，不保存完整文件内容。

主要保存：

- Connector 基本信息
- Connector 证书信息
- Connector 属性信息
- Connector ABE 用户密钥信息
- Connector 本地文件目录中的文件信息

## 2. 建表

在 Navicat 中执行：

```text
database/schema.sql
```

会创建数据库：

```text
key_management_demo
```

以及这些表：

```text
connectors
connector_certificates
connector_attributes
connector_abe_keys
connector_files
```

如果你之前已经执行过旧版 SQL，请再次执行新版 `schema.sql`，以创建新增的 `connector_files` 表。

## 3. 配置 .env

复制：

```text
.env.example
```

为：

```text
.env
```

然后配置：

```text
DB_ENABLED=true
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的数据库密码
DB_NAME=key_management_demo
DB_AUTO_INIT=false
MAX_JSON_BODY_BYTES=26214400
```

## 4. 初始化逻辑

点击前端的“重新初始化”时，系统会同时清理：

- 内存中的 Connector
- 内存中的资源和密钥
- 数据库中的 Connector 相关记录
- `storage/connectors` 中的 Connector 私有目录文件
- `storage/files` 中的系统密文文件

这样可以避免出现“前端显示为 0，但数据库仍然有旧 Connector”的不一致问题。

## 5. Navicat 中重点查看的表

### connectors

保存 Connector 基本信息。

```sql
SELECT * FROM connectors;
```

### connector_attributes

保存 Connector 属性。

```sql
SELECT * FROM connector_attributes;
```

属性变更后，旧属性会标记为：

```text
REVOKED
```

新属性会标记为：

```text
ACTIVE
```

### connector_files

保存 Connector 私有目录中的文件元数据。

```sql
SELECT * FROM connector_files;
```

你可以看到：

```text
文件属于哪个 Connector
文件在本机哪个路径
文件是本地导入还是系统下载
文件是否已经发布为系统资源
```

常见 `origin`：

```text
LOCAL_UPLOAD
SYSTEM_DOWNLOAD
```

常见 `status`：

```text
LOCAL
PUBLISHED
DOWNLOADED
```

### connector_abe_keys

保存 Connector 当前和历史 ABE 用户密钥信息。

```sql
SELECT * FROM connector_abe_keys;
```

## 6. 推荐观察流程

1. 点击“重新初始化”。
2. 手动新增 Provider 和 Consumer。
3. 查看 `connectors` 表，确认 Connector 写入数据库。
4. 导入本地文件到 Provider 目录。
5. 查看 `connector_files` 表，确认出现 `LOCAL_UPLOAD` 文件。
6. 发布该文件到系统。
7. 再查看 `connector_files`，确认该文件状态变为 `PUBLISHED`。
8. 使用 Consumer 下载系统文件。
9. 再查看 `connector_files`，确认 Consumer 多了一条 `SYSTEM_DOWNLOAD` 文件记录。

## 7. 常见问题

如果启动时报表不存在，请重新执行：

```text
database/schema.sql
```

如果数据库没有数据，请检查：

```text
DB_ENABLED=true
```

并确认修改 `.env` 后已经重新启动项目。
