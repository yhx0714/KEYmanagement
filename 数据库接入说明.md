# 数据库接入说明：使用 MySQL 保存 Connector 信息

## 1. 说明

当前项目已经支持把 Connector 信息保存到 MySQL / MariaDB 数据库中。

这里的数据库主要用于保存：

- Connector 基本信息
- Connector 证书信息
- Connector 属性信息
- Connector 对应的 ABE 用户密钥信息

这样你就可以在 Navicat 中直接观察：

```text
注册了哪些 Connector
每个 Connector 有哪些属性
属性变更后旧属性是否被撤销
ABE 用户密钥是否重新签发
```

注意：Navicat 是数据库管理工具，不是数据库本身。你需要确认 Navicat 当前连接的是 MySQL 或 MariaDB。

## 2. 第一步：安装依赖

进入项目目录：

```powershell
cd C:\Users\G3196\Desktop\生产实习\连接器的密钥管理模型
```

安装依赖：

```powershell
npm install
```

项目使用 `mysql2` 连接 MySQL。

## 3. 第二步：创建数据库表

打开 Navicat，连接你的 MySQL 数据库。

新建查询，执行项目中的 SQL 文件：

```text
database/schema.sql
```

这个 SQL 会创建数据库：

```text
key_management_demo
```

以及四张表：

```text
connectors
connector_certificates
connector_attributes
connector_abe_keys
```

## 4. 第三步：配置 .env

复制项目中的：

```text
.env.example
```

改名为：

```text
.env
```

然后修改数据库连接信息：

```text
DB_ENABLED=true
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的数据库密码
DB_NAME=key_management_demo
DB_AUTO_INIT=false
```

字段说明：

| 配置项 | 说明 |
| --- | --- |
| `DB_ENABLED` | 是否启用数据库 |
| `DB_HOST` | 数据库地址 |
| `DB_PORT` | 数据库端口，MySQL 默认 3306 |
| `DB_USER` | 数据库用户名 |
| `DB_PASSWORD` | 数据库密码 |
| `DB_NAME` | 数据库名称 |
| `DB_AUTO_INIT` | 是否启动时自动建表 |

建议初学阶段先用 Navicat 手动执行 SQL，因此：

```text
DB_AUTO_INIT=false
```

## 5. 第四步：启动项目

```powershell
npm start
```

浏览器访问：

```text
http://localhost:3000
```

点击：

```text
生成演示数据
```

系统会自动注册：

```text
Connector A
Connector B
```

同时会把它们写入数据库。

## 6. 在 Navicat 中观察数据

### 6.1 查看 Connector 基本信息

执行：

```sql
SELECT * FROM connectors;
```

你应该能看到类似：

```text
conn-001  Connector A  PROVIDER  REGISTERED
conn-002  Connector B  CONSUMER  REGISTERED
```

### 6.2 查看 Connector 属性

执行：

```sql
SELECT * FROM connector_attributes;
```

初始情况下，Connector B 可能有：

```text
department = rd
role = researcher
```

### 6.3 查看证书信息

执行：

```sql
SELECT * FROM connector_certificates;
```

可以看到每个 Connector 对应的证书编号、证书状态和指纹。

### 6.4 查看 ABE 用户密钥

执行：

```sql
SELECT * FROM connector_abe_keys;
```

可以看到每个 Connector 对应的 ABE 用户密钥。

## 7. 属性变更时数据库会发生什么

在页面点击：

```text
将 Consumer 改为销售属性
```

系统会把 Connector B 的旧属性标记为：

```text
REVOKED
```

然后插入新属性：

```text
department=sales
role=researcher
```

你可以执行：

```sql
SELECT connector_id, attr_key, attr_value, status
FROM connector_attributes
WHERE connector_id = 'conn-002'
ORDER BY id;
```

你应该能看到旧属性和新属性同时存在：

```text
department  rd          REVOKED
role        researcher  REVOKED
department  sales       ACTIVE
role        researcher  ACTIVE
```

这说明数据库已经记录了属性变化历史。

## 8. 当前数据库接入实现到了什么程度

当前已经实现：

- 注册 Connector 时写入数据库
- 查询 Connector 时优先从数据库读取
- 修改 Connector 属性时更新数据库
- 属性变更后旧属性标记为 `REVOKED`
- 新 ABE 用户密钥写入数据库
- 旧 ABE 用户密钥状态更新为 `REVOKED`

当前仍然保留在内存中的内容：

- 数据资源
- 数据密文
- DEK 数据加密密钥
- 流程日志

这是为了先把 Connector 信息持久化跑通。后续可以继续把资源、密钥和日志也迁移到数据库。

## 9. 常见问题

### 9.1 启动时报 mysql2 未安装

执行：

```powershell
npm install
```

### 9.2 启动时报数据库连接失败

检查 `.env`：

```text
DB_HOST
DB_PORT
DB_USER
DB_PASSWORD
DB_NAME
```

也可以先在 Navicat 中确认同一组账号密码能连接成功。

### 9.3 页面可以打开，但数据库没有数据

检查 `.env` 中是否启用了数据库：

```text
DB_ENABLED=true
```

然后重启项目，再点击“生成演示数据”。

### 9.4 提示表不存在

说明还没有执行建表 SQL。

在 Navicat 中执行：

```text
database/schema.sql
```

或者把 `.env` 设置为：

```text
DB_AUTO_INIT=true
```

让项目启动时自动尝试建表。
