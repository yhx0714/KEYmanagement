# 数据库接入说明：让系统具备完整持久化能力

本项目使用 MySQL 保存系统运行状态，Navicat 可以直接查看和管理这些数据。数据库只保存元数据和密钥演示数据，不保存完整明文文件；实际文件仍保存在本地目录中。

## 1. 当前持久化范围

系统现在会保存：

- Connector 基本信息、证书、属性、ABE 用户密钥
- Connector 私有目录中的文件元数据
- 已发布到系统中的资源信息
- 资源访问策略和加密文件路径
- KMS 主密钥、DEK 数据密钥、密钥状态和版本
- 上传、下载、拒绝访问、重加密、撤销等操作日志

因此重新执行 `npm start` 后，后端会自动读取数据库，恢复 Connector、资源、密钥和日志，然后继续在之前的数据基础上运行。

## 2. 建表方式

在 Navicat 中打开并执行：

```text
database/schema.sql
```

会创建数据库：

```text
key_management_demo
```

主要表包括：

```text
connectors
connector_certificates
connector_attributes
connector_abe_keys
connector_files
system_settings
resources
data_keys
access_logs
```

如果你已经执行过旧版 SQL，也需要重新执行一次新版 `schema.sql`，用于补齐新增表和字段。

## 3. .env 配置

`.env` 中建议配置为：

```text
DB_ENABLED=true
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的数据库密码
DB_NAME=key_management_demo
DB_AUTO_INIT=true
MAX_JSON_BODY_BYTES=26214400
```

`DB_AUTO_INIT=true` 的作用是启动项目时自动执行建表脚本。这个过程不会清空业务数据，只会创建不存在的表，并补充必要字段。

如果你希望完全手动管理数据库，也可以保持：

```text
DB_AUTO_INIT=false
```

但这种情况下必须先在 Navicat 中手动执行新版 `database/schema.sql`。

## 4. 重启后的恢复逻辑

项目启动时会执行：

```text
读取系统元数据
恢复 CA / AA / KMS 基础状态
读取 Connector 列表
读取 Connector 私有文件记录
读取系统资源
读取 DEK 和密钥状态
读取操作日志
恢复 ID 编号计数
进入 READY 状态
```

所以正常情况下，浏览器打开页面后不需要再点击“初始化”，可以直接看到数据库中已有的 Connector、资源和密钥。

## 5. 初始化按钮的含义

前端的“初始化”现在应理解为“重置演示系统”。

点击后会清空：

- 数据库中的 Connector、资源、密钥和日志
- `storage/connectors` 下的 Connector 私有目录文件
- `storage/files` 下的系统加密文件
- 后端当前内存状态

平时重新启动项目时不要点击初始化，只有想重新开始一轮全新演示时才使用。

## 6. Navicat 中建议查看的表

查看 Connector：

```sql
SELECT * FROM connectors;
```

查看 Connector 属性：

```sql
SELECT * FROM connector_attributes;
```

查看 Connector 私有文件：

```sql
SELECT * FROM connector_files;
```

查看系统已发布资源：

```sql
SELECT * FROM resources;
```

查看密钥状态：

```sql
SELECT key_id, key_type, status, version, owner_id, resource_id, parent_key_id, material_ref
FROM data_keys;
```

查看操作日志：

```sql
SELECT * FROM access_logs ORDER BY id DESC;
```

## 7. 测试持久化是否成功

推荐按这个流程测试：

1. 启动项目并打开页面。
2. 手动添加 Provider 和 Consumer。
3. Provider 导入本地文件。
4. Provider 发布文件资源。
5. Consumer 下载文件。
6. 停止 `npm start`。
7. 重新执行 `npm start`。
8. 刷新浏览器页面。

如果页面仍能显示之前的 Connector、文件资源、密钥状态和日志，就说明数据库持久化已经生效。

## 8. 常见问题

如果启动时报表不存在，说明数据库结构还没有更新。请在 Navicat 中重新执行：

```text
database/schema.sql
```

或者临时设置：

```text
DB_AUTO_INIT=true
```

然后重新启动项目。

如果页面数据为空，但 Navicat 中存在数据，请确认：

```text
DB_ENABLED=true
DB_NAME=key_management_demo
```

并确认项目启动时连接的是同一个 MySQL 实例和同一个数据库。
